BEM Methodology

Block-Element-Modifier: A Scalable CSS Architecture

What is BEM?

BEM (Block, Element, Modifier) is a methodology that helps you create reusable components and code sharing in front-end development.

BEM Structure:

  • Block: Standalone component that is meaningful on its own (e.g., .card, .button)
  • Element: Part of a block that has no standalone meaning (e.g., .card__title, .button__icon)
  • Modifier: Flag on a block or element to change appearance or behavior (e.g., .card--featured, .button--primary)
/* Block component */
.card {}

/* Element that depends on the block */
.card__image {}

/* Modifier that changes the style of the block */
.card--featured {}

Benefits of BEM

  • Modularity: Blocks can be reused across projects
  • Clear Structure: Easy to understand relationships between CSS and HTML
  • Low Specificity: Avoids specificity wars with flat structure
  • Team Scalability: Multiple developers can work without conflicts
  • Self-documenting: Class names reveal component structure

BEM Naming Rules:

  1. Block names can use letters, digits, and dashes: .block-name
  2. Elements are delimited with double underscore: .block__element
  3. Modifiers are delimited with double hyphen: .block--modifier
  4. Element modifiers: .block__element--modifier
  5. Never use element selectors (div, p) in CSS - only class names
