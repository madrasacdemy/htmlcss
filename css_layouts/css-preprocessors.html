Mixins

// Define a mixin
@mixin button($bg-color) {
  display: inline-block;
  padding: 8px 16px;
  border-radius: $border-radius;
  background-color: $bg-color;
  color: white;
  cursor: pointer;
  @include transition;

  &:hover {
    opacity: 0.9;
    transform: translateY(-1px);
  }
}

// Use the mixin
.btn-primary {
  @include button($primary-color);
}

Extend/Inheritance

%message-shared {
  padding: 15px;

}

.success-message {
  @extend %message-shared;

Less Features

Less Demo

This demonstrates features that would be written in Less.

Less also supports variables, mixins, nesting and more.

Less has some different syntax and features than Sass.

Variables

// Less Variables
@primary-color: #9b59b6;
@secondary-color: #3498db;
@text-color: #34495e;

.header {
  color: @primary-color;
}

Mixins

// Define a mixin
.gradient-bg() {
  background: linear-gradient(to right, @primary-color, @secondary-color);
  color: white;
}

// Use the mixin
.special-box {
  .gradient-bg();
  padding: 20px;
}

Color Functions

.lighten-bg {
  background-color: lighten(@primary-color, 10%);
}

.darken-bg {
  background-color: darken(@primary-color, 10%);
}

.transparent-bg {
  background-color: fade(@primary-color, 70%);
}

Key Differences

/* Sass/SCSS */
$variable: value;
@mixin name { ... }
@include name;
@extend %placeholder;

/* Less */
@variable: value;
.mixin() { ... }
.mixin();
&:extend(.class);
  • Sass has more advanced control directives (@if, @for, @each, @while)
  • Less can be compiled in the browser (though not recommended for production)
  • Sass has two syntaxes: SCSS (CSS-like) and Sass (indented)
  • Less has built-in JavaScript evaluation
  • Sass has larger community and more frameworks
